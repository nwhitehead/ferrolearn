# ferrolearn Test Fixtures

This directory contains golden test fixtures generated by scikit-learn for use as
oracle data in ferrolearn's integration and regression tests.

## Purpose

These fixtures implement the "oracle test" pattern: run the same computation in
scikit-learn (the reference implementation) and in ferrolearn, then compare the
outputs within a small numerical tolerance.  This ensures algorithmic correctness
independent of the specific solver or numerical path chosen in Rust.

## Fixture Format

Every fixture is a JSON file with this top-level structure:

```json
{
    "description": "Human-readable description of what this fixture tests",
    "sklearn_version": "1.x.x",
    "random_state": 42,
    "input": { ... },
    "params": { ... },
    "expected": { ... }
}
```

| Field           | Type            | Description                                        |
|-----------------|-----------------|----------------------------------------------------|
| description     | string          | What the fixture tests                             |
| sklearn_version | string          | scikit-learn version used to generate the fixture  |
| random_state    | int or null     | RNG seed used (null = deterministic without RNG)   |
| input           | object          | Input data (X, y, etc.) as nested lists            |
| params          | object          | Model / transformer hyper-parameters               |
| expected        | object          | Expected outputs from the fitted model             |

Arrays are stored row-major as nested JSON arrays.  All floats use standard
decimal notation (no scientific notation for human-readable values).

## Files

| File                        | Model / Metric         | Key expected fields                              |
|-----------------------------|------------------------|--------------------------------------------------|
| linear_regression.json      | LinearRegression       | coefficients, intercept, predictions             |
| ridge.json                  | Ridge (alpha=1.0)      | coefficients, intercept, predictions             |
| lasso.json                  | Lasso (alpha=0.1)      | coefficients, intercept, predictions             |
| logistic_regression.json    | LogisticRegression     | coefficients, intercept, predicted_classes, predicted_proba |
| standard_scaler.json        | StandardScaler         | mean, std, transformed                           |
| minmax_scaler.json          | MinMaxScaler           | data_min, data_max, transformed                  |
| robust_scaler.json          | RobustScaler           | center, scale, transformed                       |
| classification_metrics.json | Binary classification  | accuracy, precision, recall, f1, confusion_matrix |
| regression_metrics.json     | Regression metrics     | mae, mse, rmse, r2                               |
| kfold.json                  | KFold (5 folds)        | folds[i].{train, test} index arrays              |

## Using Fixtures in Rust Tests

Load a fixture with `serde_json` and `include_str!`:

```rust
#[test]
fn test_linear_regression_oracle() {
    let fixture: serde_json::Value = serde_json::from_str(
        include_str!("../../fixtures/linear_regression.json")
    ).unwrap();

    let coefs = fixture["expected"]["coefficients"]
        .as_array().unwrap()
        .iter()
        .map(|v| v.as_f64().unwrap())
        .collect::<Vec<_>>();

    // ... fit your model and compare with float_cmp or approx
}
```

Recommended tolerance for comparing floating-point values:
- Use `approx::assert_relative_eq!(a, b, epsilon = 1e-6)` for scalar outputs.
- Use `float_cmp::approx_eq!(f64, a, b, ulps = 4)` for tighter ULP-based checks.

## Regenerating Fixtures

```bash
pip install scikit-learn numpy --quiet
python scripts/generate_fixtures.py
```

Re-run this script whenever:
- scikit-learn is upgraded and you want to track the new reference values.
- New fixture types are added.
- A bug in the generation script is fixed.

Always commit both the updated `scripts/generate_fixtures.py` and the new JSON
files together so the fixtures are reproducible from the script.
